# hi-im-usrsvr M1 实施清单

> **对应生态里程碑**：档 C **M4**（与 hi-im-seqsvr 并行）；**M5** iplist、**M6** 群生命周期见文末预览  
> **工期**：约 2 周（全职）  
> **前置**：hi-im-api / hi-im-hubclient **v0.1.0**；hi-im-seqsvr **v0.1.0**（或本地联调）；hi-im M3 Hub 可冒烟

---

## 目标

交付 **用户注册 + 会话 ONLINE** 最小栈：Gin HTTP 注册、hubclient BACKEND 处理 ONLINE、gRPC 调 seqsvr、Redis 写会话。**不含** gateway WS、msgsvr 群聊 fan-out。

**M4 总验收**（档 C §11.3）：

| 项 | 标准 |
|----|------|
| HTTP | `GET /im/register` 返回 `sid>0` |
| ONLINE | Hub 路径上线后 Redis 存在 `im:sid:{sid}:attr` |
| 边界 | **不**验收 WS、iplist、群聊、GROUP-CHAT |

## 常见误区（档 C 对齐）

| 误区 | 正确 |
|------|------|
| usrsvr SUB `GROUP-CHAT` | **msgsvr** SUB；usrsvr 只处理群 **元数据** |
| M4 做 iplist + WS Demo | M4 仅 register + ONLINE；WS **M5** |
| M4 验收双窗口群聊 | 群聊 **M6**（需 msgsvr） |
| 在 usrsvr 实现 AllocSid | 调 **seqsvr** gRPC |
| 手写 Redis 键字符串 | 用 **hi-im-api/rediskey** |
| usrsvr 连 FORWARD | 业务进程走 **BACKEND**（gateway 走 FORWARD） |
